# 版本说明 v2.5.0 - 前端适配与交互优化

## 1. 版本概览
**发布日期**: 待定
**核心目标**: 实现 Web 前端对“指数分析 Workflow”的完整支持，打通从用户配置到结果展示的全链路。

## 2. 新增特性

### 2.1 指数分析模式 (Index Analysis Mode)
- **模式切换**: 用户可在“单股分析”页面一键切换至“指数分析”模式。
- **智能适配**:
    - 自动适配指数代码输入规则（支持 `.SH`, `.SZ` 等后缀）。
    - 自动筛选适用于大盘分析的 Agent 列表（屏蔽个股专用 Agent）。

### 2.2 全新的分析师阵容 (New Analyst Roster)
前端正式上线以下分析师供用户选择：
- 🌍 **宏观分析师 (Macro Analyst)**: 洞察宏观经济周期与流动性。
- 📜 **政策分析师 (Policy Analyst)**: 解读国家战略与产业政策。
- 🏗️ **行业分析师 (Sector Analyst)**: 追踪板块轮动与资金流向。
- ⚖️ **多空博弈 (Bull/Bear Debate)**: 模拟市场多空双方的深度辩论。
- 🛡️ **风险裁判 (Risk Judge)**: 提供独立的风险评估与仓位建议。

### 2.3 深度分析报告视图
- 新增 **多空辩论视图**：直观展示多方与空方的核心论点及交锋过程。
- 新增 **风险仪表盘**：清晰展示当前市场风险等级与建议仓位。
- 优化 **结构化报告**：支持宏观、政策、技术等多维度报告的分页签展示。

## 3. 变更列表

### 前端 (Frontend)
- `src/constants/analysts.ts`: 新增指数分析相关 Agent 定义。
- `src/views/Analysis/SingleAnalysis.vue`: 改造表单逻辑，支持指数模式与动态 Agent 选择。
- `src/api/analysis.ts`: 更新请求参数结构，支持 `analysis_type` 字段。
- `src/components/Report/`: 新增指数分析专用结果展示组件。

### 后端 (Backend)
- (如有必要) `app/routers/analysis.py`: 确保 API 能正确接收并处理前端传递的 `analysis_type` 和 `selected_analysts` 参数。

## 4. 升级建议
- 本次更新主要集中在前端，建议清除浏览器缓存以加载最新界面。
- 建议配合 v2.4.0 及以上版本的后端服务使用，以确保所有新 Agent 均可正常调度。
