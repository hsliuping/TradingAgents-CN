# Safe Analyst (保守风险分析师)

## 1. 角色定义
**Safe Analyst** (在代码中实现为 `conservative_debator.py`) 代表了风险管理委员会中的保守派。它的核心职责是保护资产、最小化波动性，并确保稳定、可靠的增长。它始终是房间里最谨慎的声音，优先考虑本金安全。

## 2. 核心职责
- **保护资产**：在评估交易计划时，重点关注潜在的下行风险、最大回撤和本金安全。
- **批判性审查**：识别决策中可能使公司面临不当风险的高风险要素。
- **反驳激进**：积极反驳激进（Risky Analyst）和中性（Neutral Analyst）分析师的论点，指出他们过于乐观的假设。

## 3. 输入数据
Agent 接收以下核心输入：
1. **交易员/策略顾问决策** (`trader_investment_plan` 或 `strategy_report`)：作为审查对象。
2. **多维度市场报告**：
   - 宏观经济报告 (`macro_report`)
   - 政策分析报告 (`policy_report`)
   - 板块轮动报告 (`sector_report`)
   - 国际新闻报告 (`international_news_report`)
   - 技术分析报告 (`technical_report`)
   - (在个股模式下为：市场研究、情绪、新闻、基本面报告)
3. **辩论历史** (`history`, `safe_history`)：当前的辩论上下文。
4. **对手观点**：
   - 激进分析师的最后回应 (`current_risky_response`)
   - 中性分析师的最后回应 (`current_neutral_response`)

## 4. 处理流程
1. **风险扫描**：分析所有报告，寻找不利因素（如高估值、政策不确定性、技术顶背离）。
2. **构建论点**：
   - 强调 "宁可错过，不可做错" 的原则。
   - 针对对手的乐观观点进行冷思考，例如将技术突破解释为诱多，将政策利好解释为利好出尽。
3. **生成回复**：调用 LLM 生成稳重、严谨且有说服力的辩论发言。

## 5. 输出格式
输出为结构化的 Markdown 文本，包含：
- **加粗的核心观点**
- 列表形式的风险提示
- 针对性的反驳段落

**示例输出片段**：
> **Safe Analyst**:
> **风险收益比极不划算，建议空仓观望。**
> 
> 激进派被短期的反弹蒙蔽了双眼，忽视了系统性风险。
> 1. **估值泡沫**：当前指数PE已处于历史85%分位，安全边际极低。
> 2. **外围动荡**：美联储加息预期升温，全球流动性收紧将首先冲击新兴市场。
> 
> 所谓的“入场时机”很可能是接盘侠的陷阱。我们的首要任务是活下去，而不是去赌博。

## 6. 在 Workflow 中的位置
- 位于 **Risk Management Layer (风险管理层)**。
- 与 **Risky Analyst** 和 **Neutral Analyst** 并行或轮流发言。
- 其输出将被 **Risk Judge** (Risk Manager) 用于最终决策。
