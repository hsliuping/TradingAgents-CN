# Risky Analyst (激进风险分析师)

## 1. 角色定义
**Risky Analyst** (在代码中实现为 `aggresive_debator.py`) 代表了风险管理委员会中的激进派。它的核心职责是积极寻找和倡导高回报、高风险的投资机会，强调大胆策略和竞争优势。它存在的目的是为了防止团队过于保守而错失市场爆发期的机会。

## 2. 核心职责
- **倡导进取**：在评估交易计划时，重点关注潜在的上涨空间、增长潜力和创新收益。
- **挑战保守**：直接反驳保守（Safe Analyst）和中性（Neutral Analyst）分析师的观点，指出他们的谨慎可能导致的机会错失。
- **数据支撑**：利用市场情绪、宏观利好或技术突破等数据来加强激进观点的说服力。

## 3. 输入数据
Agent 接收以下核心输入：
1. **交易员/策略顾问决策** (`trader_investment_plan` 或 `strategy_report`)：作为辩论的基础靶子。
2. **多维度市场报告**：
   - 宏观经济报告 (`macro_report`)
   - 政策分析报告 (`policy_report`)
   - 板块轮动报告 (`sector_report`)
   - 国际新闻报告 (`international_news_report`)
   - 技术分析报告 (`technical_report`)
   - (在个股模式下为：市场研究、情绪、新闻、基本面报告)
3. **辩论历史** (`history`, `risky_history`)：当前的辩论上下文。
4. **对手观点**：
   - 保守分析师的最后回应 (`current_safe_response`)
   - 中性分析师的最后回应 (`current_neutral_response`)

## 4. 处理流程
1. **分析现状**：读取所有市场报告和策略建议。
2. **构建论点**：
   - 识别策略中的进攻性元素并加以放大。
   - 针对对手（Safe/Neutral）提出的担忧（如波动性、政策风险）进行反击，例如将波动解释为交易机会，将政策风险解释为市场过度反应。
3. **生成回复**：调用 LLM 生成具有感染力、犀利且自信的辩论发言。

## 5. 输出格式
输出为结构化的 Markdown 文本，包含：
- **加粗的核心观点**
- 列表形式的论据支撑
- 针对性的反驳段落

**示例输出片段**：
> **Risky Analyst**:
> **现在的回调正是千载难逢的入场时机！**
> 
> 保守派过分强调了短期波动，却忽视了技术面上底部的强力支撑。
> 1. **资金流向逆转**：昨日北向资金大幅净流入，显示聪明钱已经进场。
> 2. **政策底已现**：最新的货币宽松信号明确，市场流动性将迎来拐点。
> 
> 对手中所谓的“等待右侧交易”只会让我们在低位踏空，现在必须果断出击！

## 6. 在 Workflow 中的位置
- 位于 **Risk Management Layer (风险管理层)**。
- 与 **Safe Analyst** 和 **Neutral Analyst** 并行或轮流发言（取决于具体的图执行逻辑）。
- 其输出将被 **Risk Judge** (Risk Manager) 用于最终决策。
