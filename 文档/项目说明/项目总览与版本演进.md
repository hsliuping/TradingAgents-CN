# TradingAgents-CN 项目总览与版本演进

## 📖 项目简介
TradingAgents-CN 是一个基于多智能体（Multi-Agent）协作的量化投资决策系统。它利用大语言模型（LLM）的分析能力，结合专业的金融数据源，模拟真实的投研团队工作流程，为用户提供深度的市场分析和投资策略建议。

本项目特别关注**指数分析Workflow**的构建，旨在解决传统量化系统缺乏深度逻辑推理、纯LLM系统缺乏实时数据支撑的痛点。

---

## 🚀 指数分析 Workflow
指数分析工作流模拟了一个专业的指数投研团队，包含以下核心角色：

1.  **Macro Analyst (宏观分析师)**: 分析GDP、CPI、PMI等宏观指标及经济周期。
2.  **Policy Analyst (政策分析师)**: 解读国家战略、财政/货币政策及产业支持政策。
3.  **Sector Analyst (板块分析师)**: 分析板块资金流向、热点主题及行业轮动。
4.  **International News Analyst (国际新闻分析师)**: 监控国际舆情、地缘政治及外部冲击。
5.  **Technical Analyst (技术分析师)**: 基于量化指标（MA, MACD, RSI等）进行趋势研判。
6.  **Bull/Bear Researchers (多空辩论组)**: 模拟市场上多头与空头的观点博弈，挖掘潜在风险与机会。
7.  **Strategy Advisor (策略顾问)**: 综合各方观点，给出最终的仓位建议、分层配置及动态调整策略。

## 📅 版本演进历史

### v2.0.0 - 基础构建 (Foundation)
**发布日期**: 2024-12-14
- **核心目标**: 构建指数分析的基础框架，实现核心Agent节点。
- **主要特性**:
    - 实现了 Macro, Policy, Sector, Strategy 四大核心Agent。
    - 集成 AKShare 数据源，支持宏观、政策、板块数据的获取。
    - 实现了基于 MongoDB 的数据缓存与降级机制。
    - 确立了线性的工作流路由机制。

### v2.1.0 - 架构优化 (Architecture Optimization)
**发布日期**: 2024-12-14
- **核心目标**: 实现**职责分离**架构，优化决策机制。
- **主要特性**:
    - **新增 International News Analyst**: 引入国际视野，弥补信息盲区。
    - **Policy Analyst 扩展**: 区分长期战略政策与短期政策利好。
    - **Strategy Advisor 重构**: 引入统一决策算法，实现双层仓位决策（基础仓位+短期调整）。
    - **输出增强**: 提供分层持仓策略（核心/战术/现金）和动态调整触发条件。

### v2.2.0 - 数据源优化与实时决策 (Data Source & Realtime)
**发布日期**: 2025-12-20
- **核心目标**: 增强数据底座，支持实盘辅助决策。
- **主要特性**:
    - **混合数据架构**: 引入 Tushare Pro 为主源，AKShare 为辅，构建高可用数据层。
    - **Technical Analyst**: 新增独立的技术分析Agent，提供纯量化的趋势研判。
    - **分时决策体系**: 支持早盘（侧重竞价/外盘）、尾盘（侧重趋势确认）、盘后（侧重复盘）三种分析模式。
    - **实时监控引擎**: 支持秒级数据穿透，适应盘中决策需求。

### v2.3.0 - 多空博弈与深度决策 (Bull/Bear Debate)
**发布日期**: 2025-12-21 (Current)
- **核心目标**: 引入多空博弈机制，提升决策的辩证深度。
- **主要特性**:
    - **多空辩论层**: 新增 Index Bull/Bear Researcher，针对目标指数进行观点交锋。
    - **决策深度优化**: Strategy Advisor 必须综合辩论双方观点，拒绝“和稀泥”。
    - **报告增强**: 策略报告包含辩论总结与胜负手分析，逻辑更具说服力。
    - **Workflow升级**: 引入循环辩论机制，确保真理越辩越明。

## 🛠️ 快速开始

### 1. 启动后端服务
确保已安装 Docker 和 Python 环境。

```bash
# 启动 Redis 和 MongoDB
docker start tradingagents-redis tradingagents-mongo

# 启动后端 API 服务
python -m app.main
```

### 2. 运行指数分析
使用提供的脚本直接调用后端接口进行分析。

```bash
# 分析半导体指数 (H30184.CSI)
python scripts/download_index_report.py --index 半导体 --depth 深度
```

### 3. 查看报告
分析完成后，报告将自动保存为 PDF 和 Markdown 格式，通常位于项目根目录下。
