# 需求
由于目前的项目仅支持针对个股进行分析，但不支持针对各个指数进行分析，例如每股的纳斯达克指数、A股的沪深300指数等。请设计一个方案，为当前项目新增指数分析的功能，

# 要求（在方案设计中请思考以下问题）
- 请思考相比个股，指数如何分析能够得出更有效的结论？也就是说如何针对指数优化langgraph的流程，针对哪部分的功能需要着重加强？如有必要可以联网搜索指数相较于个股需要着重分析哪些内容
- 如何获取指数相关的信息？通过某个指定的数据源接口还是可以实现一个爬虫模块或者依赖RSS？如有必要请联网搜索如何获得指数相关的数据，或者从github上搜索是否有可以直接使用或参考的项目或模块
- 如果有可以复用的langgraph节点可以尽量考虑复用
- 如果可以尽量包含主流的板块指数，如果有的话可以通过指数代码来选取指数，例如现在通过代码选取个股

# 注意事项
- 不需要思考前端如何改造，仅需要考虑后端如何实现改造
- 最终在项目根目录下生成一个md文件保存设计方案
- 如果有查询到的信息，请在文档中标注出数据出处（例如参考或使用了某个开源项目请标注出处，或者参考了哪些内容）
