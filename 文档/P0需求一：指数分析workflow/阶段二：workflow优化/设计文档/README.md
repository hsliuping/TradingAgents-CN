## 🔑 核心设计理念

### 双层仓位决策机制

```
长期政策支持 → 基础仓位 60%
     +
短期新闻影响 → 短期调整 +15%
     =
最终持仓建议 → 75%
```

**分层持仓策略**:
- 核心长期: 40% (不轻易变动)
- 战术配置: 35% (基于短期新闻)
- 现金储备: 25% (应对波动)

**动态调整触发**:
- 如政策官宣 → 提升至90%
- 如传闻证伪 → 降至40%

## 🆕 核心模块

### 1. International News Analyst (新增)

**职责**: 监控国际媒体,捕捉政策传闻和突发事件

**数据源**:
- NewsAPI (彭博、路透)
- Google News (免费降级)

**输出**:
- 关键新闻列表
- 短期仓位调整 (±10%-20%)

### 2. Policy Analyst (扩展)

**扩展内容**:
- 识别长期战略政策 (5-10年)
- 评估政策连续性
- 给出基础仓位建议 (40%-80%)

### 3. Strategy Advisor (重构)

**重构内容**:
- 双层仓位决策
- 分层持仓策略
- 动态调整触发条件
